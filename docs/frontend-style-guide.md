# 前端风格规范

这份文档用于统一 `job-frontend-react` 的页面风格、组件组织和接口接入方式，方便在不同窗口、不同阶段继续生成前端代码时保持一致。

## 1. 产品定位

- 产品名称：`JobBright`
- 产品类型：职位平台 / 求职平台
- 前端目标：同时支持官网展示感和业务后台可扩展性
- 当前技术栈：`React + Vite + 原生 CSS`

## 2. 视觉方向

### 2.1 整体气质

- 风格关键词：专业、可信、现代、轻盈、带一点品牌感
- 页面要有“求职平台首页”的转化感，不要写成普通后台管理页面
- 避免做成通用模板站，页面要保留明确的招聘业务语义

### 2.2 颜色规范

- 主色建议：蓝绿系、深蓝系、中性灰白系
- 禁止默认使用大面积紫色
- 背景不要纯白平铺，优先使用：
  - 渐变背景
  - 柔和光斑
  - 模糊色块
  - 分层卡片

推荐基础色板：

- 品牌主色：`#0F6FFF`
- 品牌深色：`#0D1B2A`
- 强调辅助色：`#19A974`
- 背景浅色：`#F4F7FB`
- 卡片白色：`#FFFFFF`
- 正文主色：`#1F2937`
- 次级文字：`#667085`
- 边框颜色：`#D9E2F2`

### 2.3 字体规范

- 优先使用有现代感的字体组合，不要直接退回默认系统字体气质
- 中文优先：`"PingFang SC", "Microsoft YaHei", sans-serif`
- 英文优先：`"Space Grotesk", "Inter", sans-serif`
- 标题和正文可区分字体层级，但不要过度复杂

### 2.4 圆角和阴影

- 卡片圆角：`20px - 28px`
- 按钮圆角：`12px - 16px`
- 输入框圆角：`14px - 16px`
- 阴影使用柔和大阴影，不要使用厚重黑影

## 3. 页面结构规范

### 3.1 首页类页面

首页默认采用以下结构：

1. 顶部导航
2. Hero 主视觉区
3. 核心价值区
4. 业务能力区
5. 数据/企业背书区
6. CTA 行动区
7. 页脚

### 3.2 业务页

登录后业务页建议结构：

1. 顶部导航或侧边导航
2. 页面标题区
3. 检索 / 筛选区
4. 列表区
5. 空状态 / 加载态 / 错误态

### 3.3 卡片规范

- 卡片内部层级要明显
- 标题、说明、数据、操作按钮要形成稳定布局
- 不要把所有内容堆在一个白块里

## 4. 组件风格规范

### 4.1 按钮

- 主按钮：品牌主色实底
- 次按钮：浅底色或描边
- 危险按钮：仅在删除类操作使用

按钮文案风格：

- 使用动词开头
- 例子：`进入平台`、`上传文件`、`开始检索`、`查看职位`

### 4.2 表单

- 标签清晰，不要只靠 placeholder 表达含义
- 错误提示放在字段附近
- 输入框高度、边距、字体大小统一

### 4.3 列表

- 列表项要有层级信息
- 至少区分标题、摘要、标签、操作
- 空状态需要有文案提示，不允许出现空白页面

### 4.4 提示反馈

- 成功：绿色系
- 失败：红色系
- 处理中：蓝色或中性色
- 提示框不要过大，不要遮挡主流程

## 5. 交互规范

- 首屏要有轻量动效，但不要滥用
- 悬停、点击、禁用状态必须明显
- 页面切换优先做轻量过渡，不要做花哨动画
- 文件上传、登录、注册、提交等行为必须有 loading 状态

## 6. 响应式规范

- 默认支持桌面端和移动端
- 桌面端优先宽屏展示，移动端自动折叠布局
- 不允许只做 PC 布局
- 断点建议：
  - 移动端：`< 768px`
  - 平板：`768px - 1024px`
  - 桌面：`> 1024px`

## 7. React 代码规范

### 7.1 目录建议

后续页面增多后，前端按以下方式组织：

- `src/pages`：页面
- `src/components`：通用组件
- `src/services`：接口请求
- `src/constants`：常量
- `src/hooks`：复用逻辑
- `src/styles`：全局样式和变量

### 7.2 组件原则

- 一个页面负责组合，不堆大量业务细节
- 可复用的卡片、按钮、表单块尽早拆组件
- 避免把所有逻辑继续堆在 `App.jsx`

### 7.3 状态处理

- 本地页面状态优先 `useState`
- 副作用逻辑统一在 `useEffect`
- 请求逻辑尽量抽到 `services`

## 8. 接口接入规范

- 所有请求统一从一个请求层发出
- 默认响应结构按后端当前规范处理：
  - `code === "0"` 代表成功
  - `message` 代表错误信息
  - `data` 代表业务数据

- 登录态规范：
  - 前端保存后端返回的 uuid
  - 默认请求头：`X-Access-Token`
  - 当前本地存储 key：`jobbright_access_token`

## 9. 文案规范

- 文案语气要专业、直接、可信
- 不要写成营销腔或 AI 默认话术
- 页面文案优先围绕招聘业务：
  - 职位
  - 候选人
  - 企业
  - 简历
  - 投递
  - 匹配

## 10. 后续生成页面必须遵守的规则

- 保持 `JobBright` 品牌名一致
- 沿用同一套颜色、按钮、圆角、卡片风格
- 不要随意切换成新的 UI 体系
- 不要生成传统后台蓝白表格风页面作为官网首页
- 新页面默认要补齐：
  - loading
  - 空状态
  - 错误提示
  - 移动端适配
