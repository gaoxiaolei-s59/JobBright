项目经历只提取原文明确出现的项目名称、项目角色、时间、技术栈和项目成果。
不要把公司工作经历整体当成项目；除非原文明确给出了项目名称。
如果项目描述与工作经历重复，只保留项目特有的背景、技术栈、方案、成果。
如果某个项目和相邻工作经历明显属于同一段经历，可以从该相邻上下文中补充原文明确出现过的技术栈，例如 Java、Spring Boot、MySQL、Redis、Kafka；不要补充原文没有出现的技术。
techStack 只放技术名，不放业务名、公司名、项目名；同一技术只出现一次。
description 优先保留项目背景、技术方案、个人职责和成果，不要整段复制工作经历职责。
证书奖项需要从“证书、认证、荣誉、奖项、获奖、校园经历、教育经历、英语等级、软考、竞赛、奖学金、荣誉称号、校级/院级奖项”等区域召回。
itemType 为 CERTIFICATE 或 AWARD；证书/认证/英语等级/软考用 CERTIFICATE，荣誉/奖项/竞赛/奖学金用 AWARD。
不要把技能、课程、项目名称误判为证书或奖项。
